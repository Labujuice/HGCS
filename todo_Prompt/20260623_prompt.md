### Feedback
* **問題根源**：
  - 在 [gateway.py](../gateway/gateway.py) 的 `_handle_upload_mission` 中，我們在將前端航點列表編譯為 MAVLink 任務項目時，**硬編碼強行在 `seq 0` 插入了一個 `WAYPOINT` 作為 Home 點**，並使實際任務航點（包含 `TAKEOFF`）從 `seq 1` 開始。
  - 這在 ArduPilot 中是必要的，因為 ArduPilot 強制將 `seq 0` 當作 Home 點。
  - 然而在 **PX4** 中，**`seq 0` 並非 Home 點，而是任務的第一個執行航點**。當 PX4 可行性檢查器（Feasibility Checker）發現 `seq 0` 是普通 `WAYPOINT`，而後面卻有 `TAKEOFF` 時，便會判定「起飛點不是第一個航點項目（takeoff not first waypoint item）」進而拒絕整個任務。
* **修復方案**：
  - 在 [gateway.py](../gateway/gateway.py) 的遙測 HEARTBEAT 接收端記錄車輛的 `autopilot` 類型。
  - 在上傳任務時，動態判斷是否為 PX4 飛控。如果是 PX4，則**不強行在前置插入 `seq 0` Home 點**，而是直接將 `TAKEOFF` 以 `seq 0` 上傳給飛控，這能 100% 順利通過 PX4 可行性檢驗；對於非 PX4（如 ArduPilot）則保留 `seq 0` 作為 Home 點。

  - 修改 `_handle_upload_mission` 航點序列對照邏輯：在將任務轉譯為 MAVLink 任務項目上傳時，動態讀取載具飛控類型是否為 PX4 (`MAV_AUTOPILOT_PX4`，值為 12)。

### Feedback
* **問題 1：任務上傳超時 (Timeout waiting for MISSION_ACK)**
  - **根源**：原代碼在傳送完最後一個航點 (`seq == count - 1`) 後立即 `break` 退出接收 `MISSION_REQUEST` 的循環，轉而單獨等待 `MISSION_ACK`。如果最後一個航點封包發生丟包、或者飛控寫入 Flash 較慢而重複請求該航點，Gateway 會因為不再監聽 `MISSION_REQUEST` 而忽略請求，最終導致飛控不發送 `MISSION_ACK` 並引發 Gateway 超時。
  - **修復方案**：改寫上傳等待循環，將 `MISSION_REQUEST`、`MISSION_REQUEST_INT` 與 `MISSION_ACK` 整合至同一個接收循環中。只有在收到 `MISSION_ACK` 時才退出循環；若飛控重複請求某個航點，Gateway 會持續響應，徹底消除由於 Packet Loss 或 Flash 延時引起的上傳超時 Bug。

* **問題 2：無法切入 Auto 模式**
  - **根源**：ArduPilot 的自訂模式編碼與 PX4 完全不同。舊 Gateway 代碼中 `_handle_change_mode` 僅支援 PX4 的 Main/Sub 模式位元移操作（例如 `MISSION = (sub_mode 4 << 16) | (main_mode 4 << 8)`），將此模式值發送給 ArduPilot 會被飛控直接拒絕 (Reject)，導致無法切入 Auto 模式。
  - **修復方案**：在 `_handle_change_mode` 中，根據車輛飛控類型 `msg.autopilot` 進行分支處理。如果是 PX4，繼續採用 Main/Sub 模式位移；如果是 ArduPilot，則直接發送 ArduPilot Copter 標準的模式整數值（`AUTO = 3`、`LOITER = 5`、`RTL = 6`、`LAND = 9`）。

* **問題 3：主畫面無法識別飛控品牌**
  - **修復方案**：
    - **後端**：在 `gateway.py` 接收到 `HEARTBEAT` 訊息時，即時將 `msg.autopilot`（12 為 PX4，3 為 ArduPilot）轉換為對應的品牌字串，並在每 20Hz 的 telemetry 推播封包中新增 `"autopilot": "PX4" | "ArduPilot"` 欄位。同時對 Mock Telemetry 模擬飛機默認添加 `"autopilot": "PX4"`。
    - **前端**：更新 `VehicleTelemetry` 介面類型。在頂部狀態列的 Mode 狀態字樣旁新增精美的 `autopilot-badge` 顯示飛控品牌；在右上角選單中使用者切換下拉選單項目，顯示如 `#1 (PX4)` 或 `#2 (ArduPilot)`；在地圖點擊無人機顯示的 Leaflet Popup 中，於表格內加入一列 `Autopilot` 的實時顯示。
